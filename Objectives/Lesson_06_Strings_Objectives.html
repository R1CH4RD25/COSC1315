Lesson 06 — Strings

 

Learning Objectives

By the end of this lesson, students will be able to:

Create and manipulate string literals

  • Define strings using single quotes 'text' or double quotes "text".

  • Use triple quotes (''' or """) for multi-line strings.

  • Choose appropriate quote types to include apostrophes or quotes within strings.

Access individual characters with indexing

  • Use bracket notation [] to access characters by position (e.g., name[0] gets the first character).

  • Understand that indexing starts at 0 (first character is at index 0).

  • Use negative indices to count from the end (e.g., name[-1] is the last character).

Extract substrings using slicing

  • Use slice notation [start:end] to extract portions of a string.

  • Understand that end is exclusive (not included in the result).

  • Omit start to slice from the beginning (e.g., [:5] gets first 5 characters).

  • Omit end to slice to the end (e.g., [2:] gets from index 2 to the end).

Use string methods for manipulation

  • Convert strings to uppercase with .upper() and lowercase with .lower().

  • Find substrings using .find() (returns index or -1 if not found).

  • Replace text using .replace(old, new) to create modified copies.

  • Check if a substring exists using the in operator (returns True or False).

Format strings for output

  • Use formatted string literals (f-strings) with f"text {variable} more text".

  • Embed variables and expressions directly in strings using {variable}.

  • Understand that f-strings are cleaner and more readable than string concatenation.

Recognize common pitfalls

  • Remember that strings are immutable; methods return new strings, they don’t modify the original.

  • Avoid off-by-one errors with indexing (first character is [0], not [1]).

  • Remember that slice [start:end] excludes the character at end index.

  • Understand that accessing an invalid index causes an IndexError.

Key Terms

  • String — A sequence of characters enclosed in quotes (e.g., "Hello" or 'World').

  • Indexing — Accessing individual characters in a string using [position] notation.

  • Zero-Based Indexing — The practice of counting positions starting from 0; the first character is at index 0.

  • Negative Index — A position counted from the end of a string (e.g., -1 is the last character).

  • Slicing — Extracting a substring using [start:end] notation, where end is exclusive.

  • String Method — A function that belongs to string objects (e.g., .upper(), .find(), .replace()).

  • f-string — A formatted string literal prefixed with f that allows embedded expressions using {}.

  • Immutable — Cannot be changed after creation; string methods return new strings rather than modifying the original.

  • Substring — A portion or slice of a larger string (e.g., "Pyt" is a substring of "Python").

  • IndexError — An exception raised when attempting to access an invalid string index (e.g., name[100] when the string has 5 characters).
