Lesson 09: If Statements

Video: 58:16-1:06:24 | Duration: ~8 minutes

üìö Learning Objectives

By the end of this lesson, students will be able to:

1. Understand conditional logic and decision-making in programs
  • Explain how programs make decisions using conditions
  • Recognize that if statements allow code to execute only when certain conditions are true
  • Understand that conditional logic is fundamental to creating interactive and responsive programs
  • Apply decision-making concepts to sports scenarios (eligibility checks, scoring conditions, etc.)
2. Write basic if statements with proper syntax
  • Use the correct if statement syntax: if condition:
  • Remember to include the colon (:) at the end of the if line
  • Properly indent code blocks that should execute when the condition is true
  • Understand that Python uses indentation (4 spaces) to define code blocks, not curly braces
3. Use comparison operators to create conditions
  • Use > (greater than), < (less than), >= (greater than or equal), <= (less than or equal)
  • Use == (equal to) and != (not equal to) for equality comparisons
  • Understand the difference between = (assignment) and == (comparison)
  • Apply comparison operators to sports data (scores, stats, rankings)
4. Implement if-else statements for alternative paths
  • Use else to specify code that runs when the if condition is false
  • Understand that else provides a default action or alternative path
  • Properly structure if-else blocks with correct indentation
  • Create programs that respond differently based on user input or calculated values
5. Use elif for multiple conditions
  • Use elif (else if) to check multiple conditions in sequence
  • Understand that elif allows testing multiple scenarios without nested if statements
  • Recognize that only the first true condition in an if-elif-else chain executes
  • Apply elif to categorize data (grade ranges, performance tiers, championship brackets)
6. Debug common if statement errors
  • Identify and fix missing colons after if, elif, or else
  • Correct indentation errors that cause code to run outside the conditional block
  • Fix incorrect comparison operators (using = instead of ==)
  • Recognize logic errors where conditions are in the wrong order

üîë Key Terms

If Statement
A control structure that executes a block of code only if a specified condition is true (e.g., if score > 100: print("High score!")).
Condition
An expression that evaluates to either True or False, used to determine whether code inside an if statement should execute.
Comparison Operator
Symbols used to compare values: >, <, >=, <=, == (equal), != (not equal).
Indentation
Spaces at the beginning of a line that define code blocks in Python. Code inside an if statement must be indented (typically 4 spaces).
Else Statement
A clause that provides an alternative block of code to execute when the if condition is false (e.g., else: print("Low score")).
Elif Statement
Short for "else if," it allows checking multiple conditions in sequence. Only the first true condition executes (e.g., elif score > 50:).
Boolean Expression
An expression that evaluates to True or False. Conditions in if statements are boolean expressions.
Code Block
A group of statements that are executed together, defined by indentation in Python. The indented lines after an if statement form a code block.
Control Flow
The order in which statements are executed in a program. If statements allow you to control which code runs based on conditions.
Nested If Statement
An if statement placed inside another if statement, allowing for more complex conditional logic with multiple levels of decision-making.
üí° Pro Tip: The most common mistake beginners make with if statements is forgetting the colon (:) at the end or using incorrect indentation. Remember: if condition: always needs a colon, and the code that runs when true must be indented!
